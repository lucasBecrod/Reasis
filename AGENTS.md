# AGENTS.md - Proyecto Actividades de Aprendizaje

## Información del Proyecto Actual

**Proyecto:** Reasis - Plataforma de Actividades de Aprendizaje  
**Objetivo:** Desarrollo de una plataforma educativa digital para estudiantes de Fe y Alegría  
**Fecha de Inicio:** Enero 2025  
**Estado:** Planificación y diseño inicial  

## Transición del Proyecto Anterior

**Fecha de Transición:** 12 de enero de 2025

### Proyecto Anterior (Archivado)
- **Nombre:** Consultoría de Clustering de Instituciones Educativas
- **Estado:** ✅ Completado y aprobado
- **Ubicación de archivos:** `archivado/clustering/`
- **Resultado:** Tipología exitosa de instituciones educativas mediante clustering K-means

### Nuevo Enfoque
El proyecto evoluciona de análisis estadístico hacia desarrollo de plataforma educativa digital:
- **Antes:** Análisis de datos y clustering de instituciones
- **Ahora:** Desarrollo de actividades de aprendizaje para estudiantes

## Objetivo del Nuevo Proyecto

Crear una plataforma integral que permita:

1. **Desarrollo de Actividades**: Herramientas para crear actividades de aprendizaje interactivas
2. **Gestión Estudiantil**: Seguimiento del progreso y engagement de los estudiantes
3. **Analytics Educativos**: Métricas de aprendizaje y rendimiento
4. **Colaboración**: Herramientas para trabajo en equipo y proyectos grupales
5. **Gamificación**: Sistema de logros y motivación para los estudiantes

## Arquitectura del Proyecto

### Frontend
- **Tecnología:** Flutter + FlutterFlow
- **Plataformas:** Web, iOS, Android
- **UI Framework:** Material Design 3

### Backend
- **Plataforma:** Supabase
- **Base de Datos:** PostgreSQL
- **Autenticación:** Supabase Auth
- **Storage:** Supabase Storage

### Módulos Principales
1. **Auth Module** - Autenticación por roles
2. **Activities Module** - Gestión de actividades educativas
3. **Students Module** - Experiencia y progreso estudiantil
4. **Teachers Module** - Herramientas para docentes
5. **Admin Module** - Panel administrativo

## Métricas de Éxito

### Para Estudiantes
- Engagement en actividades (>70%)
- Tiempo promedio en plataforma
- Completitud de actividades
- Satisfacción del usuario (NPS >50)

### Para Docentes
- Adopción de herramientas (>80%)
- Creación de actividades propias
- Uso de analytics para mejora
- Satisfacción con la plataforma

### Para Instituciones
- Mejora en métricas de aprendizaje
- Eficiencia operativa
- ROI de la implementación
- Escalabilidad a toda la red

## Consideraciones Técnicas

### Performance
- Carga inicial <3 segundos
- Respuesta de API <500ms
- Soporte offline básico
- Compresión de imágenes automática

### Seguridad
- Encriptación de datos sensibles
- Audit logs de acciones críticas
- Rate limiting en APIs
- Validación de inputs

### Mantenimiento
- Logging centralizado
- Monitoring de errores
- Backup automatizado
- Updates sin downtime
